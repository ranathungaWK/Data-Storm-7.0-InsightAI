# Section B — Data Cleaning

## How we handled dirty data

### Missing values

Provide a DQ summary table (generated by the silver stage):

```
Column, MissingCount
Cooler_Count, X
Outlet_Size, Y
```

### Duplicates

We removed duplicate holidays and duplicate outlet records during the silver stage.

```
Holiday duplicates removed
Outlet duplicates removed
```

### Outliers

We applied simple rules to remove impossible values:

```
Negative sales removed
Impossible coordinates removed
```

### Data quality rules enforced

```
volume_liters > 0
total_bill_value > 0
valid outlet id
valid distributor id
```

### Quarantine logic

Failed or suspicious records are written to a `quarantine/` folder with a short reason code and a CSV snapshot. The pipeline logs the count of quarantined rows.

Diagram: `records_before_cleaning` -> `cleaning` -> `records_after_cleaning` + `rejected_records` (quarantine)
